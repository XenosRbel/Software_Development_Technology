Лабораторные работы для студентов дневной формы обучения
и вечерней формы обучения, интегрированного
со средним специальным образованием 

На лабораторных работах студент должен выполнить четыре индивидуальных задания. Номер варианта индивидуального задания выдается преподавателем.

Лабораторные практикумы

В разделе «Практика» данного ЭРУД представлены два лабораторных практикума по дисциплине:

 

1.     Лабораторный практикум «Структурный анализ и моделирование в среде CASE-средства BPwin». В данном практикуме описана среда BPwin, предназначенная для разработки моделей IDEF0, DFD, IDEF3 предметной области, контрольные вопросы, варианты заданий для лабораторных работ. Знание информации, приведенной в данном практикуме, необходимо для выполнения первой, второй и третьей лабораторных работ.

2.     Лабораторный практикум «Информационное моделирование в среде CASE-средства ERwin». В данном практикуме описана среда ERwin, предназначенная для разработки модели IDEF1Х предметной области, контрольные вопросы, варианты заданий для лабораторных работ. Знание информации, приведенной в данном практикуме, необходимо для выполнения четвертой лабораторной работы.

Правила оформления отчетов по лабораторным работам

Отчеты по лабораторным работам выполняются в текстовом редакторе Word. Необходимо установить следующие размеры страницы:

Размер страницы – 21 х 29,7 см

Поле слева – 2,5 см

Поле справа – 1,5 см

Поле сверху – 2,5 см

Поле снизу – 2,2 см.

Размер шрифта Times New Roman 14, интервал между буквами обычный, интервал между строчками одинарный, выравнивание по ширине.

Объем каждого из отчетов  должен составлять 5 - 7 страниц. Файл с заданием № Х должен иметь имя ФамилияЛРХ.doc.

Лабораторная работа №1

Тема задания

Функциональное моделирование. Методология IDEF0

Для полученного варианта индивидуального задания разработать функциональную модель предметной области в среде CASE-средства AllFusion Process Modeler (BPwin) с использованием методологии IDEF0.

 

Перед выполнением лабораторной работы необходимо изучить:

-     подразд. 5.2 «Методология функционального моделирования IDEF0» Раздела 5 «CASE-технологии структурного анализа и проектирования программных средств» из раздела «Теория» ЭРУД. В данном подразделе приведены примеры IDEF0-диаграмм, теоретические основы разработки и правила синтаксиса IDEF0-диаграмм и IDEF0-моделей;

-     п. 1.1.2 «Методология функционального моделирования IDEF0», подразд. 1.2 «Инструментальная среда BPwin 2.5», п. 1.3.1 «Создание IDEF0-модели», подразд. 1.4 «Создание отчётов в BРwin 2.5», подразд. 1.5 «Последовательность основных шагов при создании IDEF0-модели предметной области» лабораторного практикума «Структурный анализ и моделирование в среде CASE-средства Bpwin», где содержатся описание среды Bрwin, правила построения в данной среде IDEF0-диаграмм и IDEF0-моделей, пример разработки IDEF0-модели, правила генерации отчетов по разработанной модели.

В лабораторной работе следует:

-      разработать IDEF0-модель предметной области, соответствующей выданному преподавателем варианту задания. IDEF0-модель должна содержать три иерархических уровня:

·          1-й уровень – контекстная диаграмма;

·          2-й уровень – диаграмма декомпозиции контекстной диаграммы;

·          3-й уровень – диаграммы декомпозиции трех блоков диаграммы 2-го уровня;

 

-     разработанную IDEF0-модель представить в виде диаграммы дерева узлов;

-     выполнить генерацию следующих отчетов в среде BPwin по разработанной IDEF0-модели:

·          отчет об ошибках. Необходимо представить сгенерированный средой Bpwin отчет об ошибках. Если в разработанной модели средой Bpwin будут найдены ошибки, необходимо их исправить и опять сгенерировать отчет об ошибках. При отсутствии ошибок необходимо представить  скриншот с соответствующим сообщением;

·          отчет об объектах. Необходимо представить сгенерированный средой Bpwin отчет об объектах разработанной модели и провести анализ данного отчета.

Лабораторная работа №4

Тема задания

Информационное моделирование. Методология IDEF1Х

Для полученного варианта индивидуального задания разработать информационную модель предметной области в среде CASE-средства AllFusion Data Modeler (ЕRwin) с использованием методологии IDEF1Х.

 

Перед выполнением лабораторной работы необходимо изучить:

-     подразд. 5.4 «Методология информационного моделирования IDEF1X» Раздела 5 «CASE-технологии структурного анализа и проектирования программных средств» из раздела «Теория» ЭРУД. В данном подразделе приведены примеры IDEF1X-моделей, теоретические основы разработки и правила синтаксиса IDEF1X-моделей;

-     лабораторный практикум «Информационное моделирование в среде CASE-средства ERwin», где содержатся описание среды ERwin, правила построения в данной среде IDEF1X-диаграмм и IDEF1X-моделей, пример разработки IDEF1X-модели в Erwin, правила генерации отчетов по IDEF1X-модели.

В лабораторной работе следует:

-     в среде CASE-средства ERwin разработать следующие уровни IDEF1X-модели предметной области, соответствующей варианту задания:

·          Логический уровень модели. В модели должны быть определены имена всех сущностей, атрибутов, связей (прямых и обратных). Должны быть определены мощности всех связей.

·          Физический уровень модели. В модели должны быть определены типы всех атрибутов и их значения по умолчанию.

 

-     выполнить генерацию отчетов в среде ERwin по разработанной IDEF1X-модели;

-     доказать, что каждая сущность логической модели находится в третьей нормальной форме.
